# 案例：通知页精准回复某条评论

## 目标

在「通知 → 评论和@」列表里，锁定某一条评论（按用户名+评论内容+时间），在该条上点「回复」并输入发送，回复对到该条、不串行；不点用户头像（不进用户首页）。

## 前置

- 页面：先打开通知页「评论和@」：`https://www.xiaohongshu.com/notification?channelTabId=mentions`。
- 已登录且列表中有评论。

## 步骤（按顺序执行）

1. **进入通知页并拿到当前 snapshot**  
   - 操作：navigate 到上述 URL，等 1～2 秒后 snapshot（compact: true 可先看全貌）。  
   - 后续所有「选哪一条、点哪个」都基于这一份 snapshot，不跨 snapshot 用 ref。

2. **在 snapshot 里锁定要回复的那一条**  
   - 在 snapshot 里用「谁 + 评论内容片段 + 时间」唯一确定一条（例如：Wedi（Root）、「别纠结出厂序列…」、18分钟前）。  
   - 只认这一条；若多条相似用更长片段或时间区分。

3. **在该条内点「回复」（二选一，优先 B）**  
   - **A. 用 ref**：在该条对应结构里找文案为「回复」的 link/button 的 ref，用 act click 该 ref。不要用用户头像/用户链接的 ref。  
   - **B. 用 JS**：用 act 的 eval/script 能力，在页面里先找到包含该用户名/评论内容的节点，再在其父容器内找文本为「回复」的可点击元素并 click。**必须判空**：对任何可能为 null 的变量先 if 再调 querySelectorAll 等，否则易报错；若 JS 报错则退回 A。  
   - 点完后 snapshot，确认出现回复输入框或跳转到回复场景。

4. **输入回复并发送**  
   - 先 click 评论输入框聚焦，再 type 回复内容，再在当前 snapshot 里找「发送」按钮 ref 并 click。  
   - 若输入框不可见：先滚动再 snapshot，用新 ref 再点、再输入、再发。

5. **确认与下一条**  
   - snapshot 确认评论列表里出现刚发的内容或条数变化。  
   - 若继续回复下一条：navigate 回通知页，再 snapshot，在新 snapshot 里重复 2～4，**每条都用当前 snapshot 的 ref**。

## 成功标志

- 回复输入框出现且已发送；评论列表或计数能看到刚发的回复。

## 对应 SKILL 章节

- 第 6 节「在通知页面精准回复用户评论」；6.2 锁定一条、6.3 用 JS 或 ref 点回复、6.5 输入发送、6.3.2 兜底（JS 报错时用 ref）。

## 禁止

- 禁止点用户头像/用户链接（会进用户首页）。  
- 禁止用其他行的 ref 点回复（易串行）。  
- 执行过程中禁止关闭 gateway。

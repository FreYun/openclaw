# Dashboard 空白 — 逐步排查

服务器端已确认正常（根路径、JS、CSS 均 200）。按下面顺序在**浏览器**里排查。

---

## 步骤 1：禁用缓存后看 Network

1. 打开 **Chrome**，按 **F12**。
2. 切到 **Network（网络）** 面板。
3. **勾选「Disable cache」**。
4. 地址栏输入（整行复制）：
   ```
   http://127.0.0.1:18789/?token=cf5c3f0f245db51357171803cb0f6a0224b53350bc224d11
   ```
   回车。

5. 在 Network 列表里确认：
   - 第一个请求（`127.0.0.1` 或空白名）：状态必须是 **200**。
   - 有 `index-xxxxx.js`：状态必须是 **200**。
   - 有 `index-xxxxx.css`：状态必须是 **200**。

**若任一为 404 或红色** → 说明浏览器拿到的和命令行不一致，多半是缓存或旧数据。做**步骤 2**。

**若全是 200** → 做**步骤 3**。

---

## 步骤 2：彻底清掉该站点的数据

1. 在地址栏输入：`chrome://settings/siteData`
2. 在「搜索网站」里输入：`127.0.0.1`
3. 点该站点右侧的**垃圾桶**删除。
4. 关掉**所有**打开 `127.0.0.1:18789` 的标签页。
5. **新开一个标签**，再访问带 token 的地址（同上）。
6. 再看一次 Network：是否全部 200？页面是否仍空白？

---

## 步骤 3：看 Console 里所有输出

1. F12 → **Console（控制台）**。
2. 刷新页面（F5）。
3. 看是否有**任意**输出（不限于红色）：
   - **红色**：报错（记下完整内容）。
   - **黄色**：警告（例如 CSP、跨域）。
   - **灰色**：一般日志（若有 "Failed to fetch"、"CSP"、"Refused to load" 也记下）。

把看到的**第一行/最显眼的一两条**记下来。

---

## 步骤 4：看 Elements 里是否有内容

1. F12 → **Elements（元素）**。
2. 展开 `<body>`。
3. 看里面除了 `<openclaw-app></openclaw-app>` 之外，**是否还有别的节点**（例如 div、input、按钮）。

- **若只有空的 `<openclaw-app></openclaw-app>`** → 说明 JS 没把界面挂载进去（可能运行时报错或条件不满足）。
- **若里面有很多子节点** → 界面其实渲染了，可能被 CSS 藏起来（例如全白、透明、z-index 或定位问题）。

---

## 步骤 5：换浏览器 / 无痕模式

1. 用 **Edge** 或 **Firefox**（或 Chrome **无痕窗口**，Ctrl+Shift+N）。
2. 直接访问：
   ```
   http://127.0.0.1:18789/?token=cf5c3f0f245db51357171803cb0f6a0224b53350bc224d11
   ```
3. 看是否仍空白。

- **若在另一浏览器/无痕里能正常显示** → 多半是原浏览器扩展或缓存问题。
- **若仍然空白** → 继续步骤 6。

---

## 步骤 6：试「无 token」地址

1. 新开标签，只访问：`http://127.0.0.1:18789/`（不带 `?token=...`）。
2. 看是否出现**任何**界面（例如「请输入 token」「连接网关」、设置图标等），哪怕很小或只有一行字。

- **若有** → 在页面上找到输入框或设置，把 token 贴进去保存，再刷新或重开带 token 的地址。
- **若无** → 和带 token 时一样完全空白，则更可能是 JS 在两种情况下都没挂载成功。

---

## 步骤 7（最后）：降级 OpenClaw

若以上都做过仍空白，可降级到已知在 Windows 上 dashboard 正常的版本：

```powershell
# 在跑 gateway 的终端里先 Ctrl+C 停掉
npm uninstall -g openclaw
npm i -g openclaw@2026.2.21-2
openclaw gateway
```

再用浏览器访问带 token 的地址。

按 1→2→3→4 做一遍后，回复：

1. **步骤 1**：三个请求（文档、js、css）在 Network 里是 200 还是 404？
2. **步骤 3**：Console 里有没有红色/黄色/灰色里和错误、CSP、fetch 相关的内容？有的话贴第一句。
3. **步骤 4**：`<openclaw-app>` 里面是空的，还是已经有很多子节点？
